## 2. 치명적 함정 — Papago 정면 대결 불가

방한 외국인 메뉴판 번역은 **Papago의 main use case**. 10년 쌓인 한국어 카메라 번역 정확도, 오프라인 모드, 존댓말 토글, 무제한 실시간 카메라 모드를 가진 상대. **단순 번역 앱으로 MenuLens가 Papago를 이길 수 있는 시나리오는 0**. 그대로 만들면 "Papago 열화판"이 됨.

**그럼 무엇이 남는가** — Papago·Google Translate·Google Lens가 모두 **못 하는** 4개 레이어:

1. **공식 표준 번역 대조** — Papago가 "삼겹살 → pork belly"로 번역하지만, 한식진흥원 공식 표기는 *Samgyeopsal (grilled pork belly)* 고유명사 유지. 심사위원(KTO)이 정책적으로 선호하는 방향.
2. **가격 적정성** — Papago는 번역만, 이게 *이 지역 평균가 대비 얼마나 비싼지* 판단 못함. 외국인 바가지 방지가 2026년 국가 어젠다인데 이 축이 비어있음.
3. **알레르기·종교·비건 필터** — Papago는 재료 인식 0. 무슬림·셀리악·비건 관광객 세그먼트가 완전 무방비.
4. **문화 맥락 + 주문 가이드** — Papago는 "이건 어떻게 먹는 음식인지" 설명 없음. 주문할 때 "하나 주세요"를 어떻게 발음하는지도 없음.

**포지셔닝 재정의**: "MenuLens는 Papago를 대체하지 않습니다. Papago가 읽은 다음 *먹을지 말지 결정*하는 4초를 돕습니다."

## 3. MVP 재설계 — "한 메뉴, 한 결정"

기존 7개 에이전트·5개 기능 계획을 **3개 에이전트·1개 핵심 기능 + 2개 부가 레이어**로 축소.

### 한 화면 UX

```
┌─────────────────────────────────────┐
│  [메뉴판 사진]                        │
│  ┌───────────────────────────┐       │
│  │ 김치찌개 ₩12,000          │       │
│  │ Kimchi-jjigae            │       │
│  │ (fermented kimchi stew)   │       │
│  └───────────────────────────┘       │
│                                      │
│  ⚠ ₩12,000 (서울 평균 ₩8,577)       │
│    → 40% 높음. 관광지 가격일 수 있음   │
│                                      │
│  🌾 글루텐 無  🐷 돼지고기 포함         │
│  ✅ 할랄 불가  🥬 비건 불가           │
│                                      │
│  💬 "이건 숙성된 김치에 돼지고기를..."  │
│                                      │
│  🔊 [한국어 주문하기] ▶              │
│     "김치찌개 하나 주세요"             │
│                                      │
│  🟢 번역 공식 대조됨                   │
│  🟡 가격: 소비자원 참가격 기준          │
└─────────────────────────────────────┘
```

### 3개 에이전트로 압축

- **Agent 1 (Menu Reader)**: 메뉴판 이미지 → 메뉴명·가격 리스트 추출. Gemini Flash 2.5 (1차 저비용 OCR) + Claude Sonnet Vision (2차 레이아웃 보정)
- **Agent 2 (Dish Profiler)**: 메뉴명 → 한식진흥원 DB 대조 + 재료 추출 + 알레르기·종교·비건 분류 + 문화 한 줄 설명. Claude Sonnet + RAG(pgvector)
- **Agent 3 (Price Sentinel)**: 메뉴명 → 한국소비자원 참가격 8대 품목 매칭 → 적정성 판정 + TourAPI 지역 보정. 매칭 안 되면 Claude 상식 기반 범위 추정 (신뢰도 낮음 배지)

### 활용되는 9가지 AI 기술

| 기술 | 역할 | 구현 |
|---|---|---|
| Multimodal Vision LLM | 메뉴판 이미지 → 구조화 텍스트 | Claude Sonnet 4.6 Vision / Gemini 2.5 Flash |
| Machine Translation | 6개 언어 번역 | Claude Sonnet + DeepL 폴백 |
| RAG (Vector Search) | 한식진흥원 800선 대조 | pgvector on Supabase |
| Classification LLM | 재료 → 알레르기 14종 태깅 | Claude Sonnet structured output |
| Embedding 유사도 | "비슷한 다른 메뉴" 추천 | OpenAI text-embedding-3-small ($0.02/1M tokens) |
| Structured Output | JSON 스키마 UI 일관성 | Claude tools / Gemini schema |
| Tool-calling Agent | TourAPI·참가격 호출 | LangChain 또는 Claude Agent SDK |
| TTS | 한국어 주문 발음 | Google Cloud TTS 또는 CLOVA Voice |
| Prompt Engineering | 문화 맥락 설명 톤 | Few-shot + persona prompting |

**AI 스택 다양도가 처음 기획 그대로 유지된다**는 점이 중요. JH의 학습 목표와 일치.

## 6. PoC 코드 스켈레톤 (D2~D6 구현 가이드)

```python

from anthropic import Anthropic
from google.generativeai import GenerativeModel
import httpx, asyncio

anthropic = Anthropic()
gemini = GenerativeModel("gemini-2.5-flash")

# ==================== Agent 1: Menu Reader ====================
async def read_menu(image_bytes: bytes) -> list[dict]:
    """1차: Gemini Flash로 저비용 OCR"""

        {"mime_type": "image/jpeg", "data": image_bytes}
    ])
    return parse_json(r.text)

    """한식진흥원 DB 대조 + 재료·알레르기·문화 설명"""
    # RAG: pgvector에서 표준 번역 조회

    # Claude로 구조화 분석

        max_tokens=800,

        messages=[{"role": "user", "content": f"""
        메뉴: {dish_name}
        공식 번역 후보: {official}
        사용자 언어: {user_lang}
        
        다음을 JSON으로 반환:
        - translation: 공식 표기 우선, 없으면 LLM 번역
        - ingredients: 주재료 리스트
        - allergens: 14개 중 해당 (pork, beef, gluten, egg, dairy, ...)
        - halal_safe: bool
        - vegan_safe: bool  
        - culture_note: 한 문장 문화 맥락 (user_lang로)
        - pronunciation: "OOO 하나 주세요" (한글 + 로마자)
        """}]
    )
    return msg.content[0].input

# ==================== Agent 3: Price Sentinel ====================
CONSUMER_PRICE = {  # 한국소비자원 참가격 8품목 (서울 기준, 월별 스크랩)
    "김밥": 3700, "칼국수": 9846, "김치찌개": 8577, "삼계탕": 18000,
    "냉면": 12100, "삼겹살": 21500, "비빔밥": 11200, "자장면": 7692
}
async def judge_price(dish_name: str, price: int, lat: float, lon: float) -> dict:
    """가격 적정성 판정 + 지역 보정"""
    benchmark = match_consumer_price(dish_name)  # 8개 품목 중 가장 유사
    if benchmark:
        ratio = price / benchmark
        return {
            "benchmark": benchmark,
            "ratio": ratio,
            "verdict": "정상" if ratio < 1.1 else "약간 비쌈" if ratio < 1.3 else "바가지 의심",
            "source": "한국소비자원 참가격 (서울 기준)",
            "confidence": 0.85
        }
    # 미매칭 시 Claude 상식 추론 (낮은 신뢰도)
    return await claude_estimate_price(dish_name, price)

# ==================== Orchestrator ====================
async def analyze_menu(image_bytes: bytes, user_lang: str, location):
    items = await read_menu(image_bytes)
    tasks = [
        asyncio.gather(
            profile_dish(item["name"], user_lang),
            judge_price(item["name"], item["price"], *location)
        ) for item in items
    ]
    results = await asyncio.gather(*tasks)
    return [merge(item, profile, price) for item, (profile, price) in zip(items, results)]
```

**이 스켈레톤으로 실제 메뉴판 1장 → 5초 이내 구조화된 응답**까지 도달 가능. D2~D6에 이 골격을 완성하면 공모전 제안서에 "작동하는 PoC" 첨부 가능.
