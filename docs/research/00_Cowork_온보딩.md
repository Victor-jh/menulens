## 🧠 이 프로젝트에 오기까지의 의사결정 흐름

### 단계 1: 공모전 참가 결정 (최초 망설임)
과거 JH의 12개 미완 프로젝트 패턴(이디야 레시피 앱, MVNO 사업화, hwpx-studio 배포 등)에 대한 우려가 있었음. 그럼에도 참가를 결정한 이유:
- 개인 참가 가능 → 겸업 제약 없음
- 2,000만 방한객 시대 + 광장시장 소송 + 바가지 국가 어젠다가 **2026년에 처음 동시 성립**한 타이밍
- **포트폴리오·학습·진흥원 전환 레퍼런스** 확보가 본 목적 (수상은 보너스)

**진행 조건 3가지**:
1. D7 Hard Gate — PoC 미작동 시 포기
2. 기능 추가 금지 선언
3. 5월 예심 결과 후 6~9월 개발 Go/No-Go 재결정

### 단계 2: 초기안 KOMPASS(7 에이전트) → MenuLens(3 에이전트)
최초에는 외국인 방한객 전체를 타깃으로 하는 "KOMPASS — 7개 에이전트 멀티 AI 시스템"을 설계했음. 30개 시선으로 검토 결과:
- 7개 에이전트는 관광 QA 도메인에 과잉 설계
- 토큰 비용 15배, 응답 지연 30초+
- 심사위원이 "멀티 에이전트" 피로감 보유
- "한 가지에 집중한 작품"이 수상 패턴

→ **3 에이전트로 축소, 메뉴판 한 장에만 집중**

### 단계 3: "킥" 탐색 (8개 후보 → 색깔 3종 UX 선택)
"번역 + 가격 + 알레르기"는 덧셈일 뿐 킥이 아니라는 판단. 8개 킥 후보 탐색:
1. ① Voice Order Bridge (양방향 통역)
2. ② Taste DNA (취향 학습)
3. ③ Dual Price Window (외국인/내국인 가격 대조)
4. ④ K-Scene Linker (드라마 촬영지)
5. ⑤ Local Index (현지인 비율 점수)
6. ⑥ Dish to Menu (사진→메뉴판 역검색)
7. ⑦ Restaurant Dashboard (B2B)
8. ⑧ Taste Journey (문화 번역)

최종 선택: **"읽지 마세요. 초록색만 주문하세요"**
- 읽기를 없앤다 (언어·나이·시력 초월)
- 색깔 3종 UX (🟢🟡🔴)
- 탭 한 번 → TTS 주문

### 단계 4: 구현 스콥 최종 조정
- 라이브 AR 모드는 비용·복잡도 폭증 → **"찰칵 한 번" 사진 모드**
- 7개 기능 → 3개 핵심 (사진·색깔·TTS)
- Papago 정면 대결 불가 → "Papago가 읽은 다음 결정하는 4초를 돕는다" 포지셔닝

## 🏗️ 현재 아키텍처 (최종 확정)

### 3개 AI 에이전트
1. **Menu Reader** (Gemini 2.5 Flash Vision) — 메뉴판 이미지 → 메뉴명·가격 JSON
2. **Dish Profiler** (Claude Sonnet + pgvector RAG) — 한식진흥원 800선 대조 + 재료·알레르기·문화
3. **Price Sentinel** (Rule + Claude) — 한국소비자원 참가격 8품목 대조 → 🟢🟡🔴 판정

### 공공데이터 6개 교차
- TourAPI (필수, 다국어 7개, LOD SPARQL 최초 활용)
- 한식진흥원 길라잡이 800선 (공공데이터포털 15129784)
- 한국소비자원 참가격 외식비 (price.go.kr)
- 식약처 식품영양성분DB (15127578)
- 카카오 로컬 API
- KMF 할랄 인증

### 기술 스택
- Frontend: Next.js + PWA
- Backend: FastAPI (Python 3.11) on Cloud Run
- DB: Supabase Postgres + pgvector
- AI: Claude Sonnet 4.6 + Gemini 2.5 Flash + OpenAI 임베딩
- TTS: Google Cloud TTS 무료 티어
- 비용: 공모전 기간 전체 약 5만원 이하

## 🎨 UX 핵심 원칙

1. **온보딩 60초**: 알레르기·종교·언어 한 번 설정, 영구 적용
2. **앱 실행 = 카메라**: 홈화면·메뉴 없음
3. **사진 1장 → 2~3초**: 로딩 애니메이션
4. **색깔만 보기**: 🟢 정상·안전 / 🟡 주의 / 🔴 위험 / ⚪ 정보 없음
5. **탭 한 번 = 주문**: 한국어 TTS 재생, 휴대폰을 카운터에 내밈

**심사 메시지**: "결정 시간 30~60초(Papago) → 3~5초(MenuLens)"
