## 0. 이 문서의 목적

한식진흥원 "한식 길라잡이 800선" 데이터를 Supabase Postgres + pgvector에 적재하기 위한 **데이터 스키마·정규화 규칙·알레르기 태그 매핑 스펙**을 정의한다.

## 2. Supabase Postgres 스키마

### 2.1 테이블 정의

```sql
-- pgvector 확장 활성화 (D3 최초 1회)
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE hansik_800 (
    id              SERIAL PRIMARY KEY,
    source_row_no   INTEGER NOT NULL,                    -- 원본 CSV 행번호
    category_major  TEXT NOT NULL,                       -- 주식류/부식류/후식류/기타
    category_minor  TEXT NOT NULL,                       -- 찌개/구이/무침 등
    name_ko         TEXT NOT NULL,                       -- 김치찌개
    name_en         TEXT NOT NULL,                       -- Kimchi-jjigae
    name_en_desc    TEXT,                                -- fermented kimchi stew
    name_ja         TEXT,                                -- キムチチゲ
    name_zh_simp    TEXT,                                -- 泡菜汤
    name_zh_trad    TEXT,                                -- 泡菜湯
    desc_ko         TEXT,
    desc_en         TEXT,
    main_ingredients    TEXT[],                          -- ['김치','돼지고기','두부']
    allergy_tags    TEXT[],                              -- ['pork','gluten'] (§3 매핑)
    vegan_status    TEXT CHECK (vegan_status IN ('vegan','vegetarian','non_veg','unknown')),
    halal_status    TEXT CHECK (halal_status IN ('halal_likely','haram','unclear')),
    spicy_level     INTEGER CHECK (spicy_level BETWEEN 0 AND 3),  -- 0=안매움, 3=매우매움
    image_path      TEXT,
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW(),

    -- 임베딩 (§4)
    embedding_ko    VECTOR(1536)                          -- OpenAI text-embedding-3-small
);

-- 유사도 검색 인덱스 (HNSW)
CREATE INDEX hansik_800_embedding_ko_idx
    ON hansik_800 USING hnsw (embedding_ko vector_cosine_ops);

-- 텍스트 검색 인덱스 (정확 매칭 · fallback)
CREATE INDEX hansik_800_name_ko_trgm_idx
    ON hansik_800 USING gin (name_ko gin_trgm_ops);
```

### 2.2 인덱스 선택 근거
- **HNSW**: pgvector 0.5+ 기본, Recall@10 > 0.95, latency < 50ms (800행 규모)
- **trgm (trigram)**: 메뉴명 부분일치/오타 보정 (예: "김찌" → "김치찌개")
- **향후 확장 시**: 영·일·중 임베딩 추가 고려 — 지금은 한국어만 (Menu Reader가 한국어로 뽑아줌)

## 3. 알레르기 태그 14종 매핑 규칙

### 3.1 태그 정의 (ADR-002 기반 확장)

| 태그 | 영문 | 매핑 조건 (재료 키워드) |
|---|---|---|
| `pork` | Pork | 돼지, 삼겹, 항정, 목살, 제육, 족발, 순대, 햄, 베이컨 |
| `beef` | Beef | 소, 쇠고기, 갈비, 육회, 양지, 차돌, 우둔 |
| `chicken` | Chicken | 닭, 계, 치킨, 삼계, 봉, 다리 |
| `seafood` | Seafood | 해물, 새우, 오징어, 문어, 낙지, 조개, 꽃게, 전복, 굴, 홍합 |
| `fish` | Fish | 생선, 고등어, 갈치, 멸치, 참치, 연어, 도미, 조기, 명태 |
| `shellfish` | Shellfish | 새우, 게, 꽃게, 대게, 조개, 홍합, 전복 |
| `egg` | Egg | 계란, 달걀, 알, 지단 |
| `dairy` | Dairy | 우유, 치즈, 버터, 요거트, 크림, 분유 |
| `gluten` | Gluten | 밀, 면, 국수, 빵, 부침, 전, 튀김옷, 만두피, 칼국수, 짜장 |
| `soy` | Soybean | 콩, 두부, 된장, 간장, 청국장, 비지 |
| `nuts` | Tree Nuts | 호두, 잣, 아몬드, 피칸, 캐슈, 밤 |
| `peanut` | Peanut | 땅콩 |
| `sesame` | Sesame | 참깨, 참기름, 깨소금 |
| `alcohol` | Alcohol | 술, 소주, 맥주, 청주, 정종, 미림, 맛술, 와인 |

**규칙**:
- `main_ingredients` 배열에 키워드가 하나라도 포함되면 해당 태그 자동 추가
- 대소문자·띄어쓰기 무시 (`LIKE '%키워드%'` 정규화)
- 복수 태그 허용 (예: 김치찌개 → `['pork','soy','gluten(?)']`)
- `?` 의심 케이스는 UI에서 ⚪(정보없음) 처리

### 3.2 비건/베지테리언/할랄 자동 분류

**`vegan_status`**:
- `vegan`: 고기·해물·생선·계란·유제품 태그 모두 없음
- `vegetarian`: 고기·해물·생선 태그 없음 (계란·유제품은 허용)
- `non_veg`: 위 외 전부
- `unknown`: 재료 정보 부족

**`halal_status`**:
- `haram`: `pork`, `alcohol` 중 하나라도 포함
- `halal_likely`: 위 해당 없음 + 수동 확인 필요
- `unclear`: 미림·조미료 등 숨은 알코올 의심 재료 존재

**주의**: `halal_likely`는 **KMF 인증과 다름**. 제안서·UI에서 "할랄 인증 = KMF 목록 교차 확인"으로 분리 명시.

### 3.3 매운맛 강도 (`spicy_level`)

| 레벨 | 의미 | 자동 분류 키워드 |
|---:|---|---|
| 0 | 안 매움 | 기본 |
| 1 | 약간 매움 | 고춧가루, 고추장 소량 (범위 판단 어려움 → 수동) |
| 2 | 매움 | 김치, 김치찌개, 제육볶음 |
| 3 | 매우 매움 | 불닭, 낙곱새, 해물찜 매운맛, 매운탕 |

**한계**: 자동 규칙으로는 정확 분류 불가. Claude Sonnet에 `desc_ko` 전달 → structured output으로 분류 요청 (D5 작업).

## 4. 임베딩 생성 규칙

### 4.1 대상 텍스트 조합

`embedding_ko` 생성 시 아래 형식으로 한 문장 합성 후 임베딩:

```
{name_ko} {name_en} {category_minor} {main_ingredients}
예: 김치찌개 Kimchi-jjigae 찌개 김치 돼지고기 두부 양파
```

**근거**:
- 메뉴명만으로는 희소 (800건, 어휘 다양성 부족)
- 영문·카테고리·재료를 포함하면 다국어 검색 폴백 가능
- OpenAI `text-embedding-3-small` (1536차원, $0.00002/1K tokens)

### 4.2 임베딩 예산
- 800건 × 평균 30토큰 = 24,000 토큰
- $0.00048 = **0.5원 미만** (1회 적재 비용)
- 재실행 방지: `updated_at` 기준 diff 감지 후 변경분만 재임베딩

### 4.3 유사도 검색 임계값
- Menu Reader가 뽑은 메뉴명 → pgvector 유사도 검색
- `cosine_distance < 0.35` 매칭 허용
- `0.35 <= distance < 0.50` → 후보 3개 제시 + Claude Sonnet 최종 선택
- `distance >= 0.50` → 매칭 실패, ⚪(정보없음) 처리

## 5. 정규화 규칙

### 5.1 메뉴명 정규화 (`name_ko`)
- 공백 제거 ("김치 찌개" → "김치찌개")
- 특수문자 제거 (괄호·따옴표)
- 한글 자모 분리 방지 (NFC 정규화)
- 숫자 제거 ("김치찌개2인분" → "김치찌개")
- 가격·단위 제거 ("김치찌개 8,000원" → "김치찌개")

### 5.2 재료 정규화 (`main_ingredients`)
- 쉼표 기반 split
- 각 항목 trim + 공백 단일화
- 빈 문자열 제거
- 한글 불용어 제거 ("등", "기타", "약간")

### 5.3 영문 표기 정규화 (`name_en`)
- 한식진흥원 공식 표기 **그대로 유지** (임의 수정 금지)
- 변경 필요 시 DECISIONS.md에 ADR 작성 후 반영

## 6. 적재 파이프라인 (맥북 Claude Code 실행)

### 6.1 단계

```python
# backend/scripts/load_hansik_800.py (맥북이 작성)

# 1. CSV 다운로드 (최초 1회)
# 2. pandas로 로드

# 4. 정규화 (§5)
# 5. 알레르기 태그 매핑 (§3.1)
# 6. vegan/halal/spicy 분류 (§3.2, 3.3)
# 7. 임베딩 생성 (§4.1)
# 8. Supabase upsert (§2.1 스키마)
# 9. 인덱스 재생성 확인
# 10. Canary 쿼리: "김치찌개" 검색 → 단일 결과
```

### 6.2 검증 쿼리 (D3 완료 기준)

```sql
-- 1. 총 건수
SELECT COUNT(*) FROM hansik_800;  -- 800 근방

-- 2. 알레르기 태그 분포
SELECT allergy_tags, COUNT(*) FROM hansik_800
GROUP BY allergy_tags ORDER BY 2 DESC LIMIT 10;

-- 3. 유사도 검색 샘플
SELECT name_ko, name_en, 1 - (embedding_ko <=> query_vec) AS similarity
FROM hansik_800, embed_query('김치찌개') AS query_vec
ORDER BY embedding_ko <=> query_vec LIMIT 5;

-- 4. 할랄 분포
SELECT halal_status, COUNT(*) FROM hansik_800 GROUP BY 1;
```

### 6.3 완료 기준 (ROADMAP D3)
- [ ] 800건 ±10% 범위 적재
- [ ] 캐너리 쿼리 "김치찌개" → `Kimchi-jjigae (fermented kimchi stew)` 반환
- [ ] 알레르기 태그 14종 중 최소 10종 이상 실제 데이터에 등장
- [ ] 비건·할랄 분류 각 카테고리 최소 10건 이상 분포
- [ ] 임베딩 검색 응답 시간 < 100ms (p95)
