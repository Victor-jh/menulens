> D10 제안서 작성 시 이 파일을 참조.

## Persona 1: 천이야 (Chen Yiya) — 대만 FIT, 25세, 1회차 방한

**배경**: 대만 타이베이에서 5일 관광. K-드라마 팬. 한국어 몰라요. 영어는 중급.

**현재 경로** (MenuLens 없이):
1. 구글 지도로 검색 → 한국 지도 길찾기가 안 돼서 당황
2. 네이버 지도 앱 설치 → 한국어라 읽기 어려움
3. 캐치테이블 설치 시도 → 한국 전화번호 필요, 포기
4. 호텔 컨시어지에 의존 → 호텔 근처만 맴돔

**Pain Point**: "내가 가고 싶은 동네 식당에 들어가야 하는데, 메뉴판을 못 읽어서 무서워. 바가지 당할까 봐 더 무서워."

**MenuLens가 주는 해결**:
- 메뉴판 찰칵 → 중국어 번체로 번역 + 대만 시세와 비교 가능
- 🟢 메뉴만 탭 → 한국어 주문 자동 재생
- 설치 없이 **QR로 접속 가능** (PWA)

**사용 시나리오**:
> "홍대 골목 식당 앞. 메뉴판이 전부 한글. MenuLens 열기 → 찰칵 → 3초 뒤 초록색 메뉴 3개 표시. 제일 싸고 초록색인 '된장찌개' 탭 → '된장찌개 하나 주세요' 재생 → 직원에게 휴대폰 내밂 → 주문 완료."

## Persona 2: 타나카 유이 (Tanaka Yui) — 일본 30대, 3회차 재방문자, **Pescatarian (어식주의자)**

**배경**: 도쿄 거주. 한국 여행 3번째. 이번엔 서울 연신내·종로 일대 5일 체류. 한국어 인사말 정도. 어식주의(생선·해물 OK, 육류 NO)로 일본 내에서도 외식 옵션이 까다로움.

**현재 경로**:
1. Papago 카메라 번역 → 재료까지 안 알려줌 ("불닭갈비 → spicy chicken ribs"는 알아도 "닭이 들어있는지" 식별은 사용자 몫)
2. 일본어 블로그·유튜브로 사전 조사 → 3시간 투자
3. 현지에서는 "본 것만" 또는 "확실한 회·생선 메뉴만"으로 좁힘

**Pain Point**: "재방문이라 로컬 맛집을 찾고 싶지만, 어식주의라 김밥이나 분식조차 메뉴마다 일일이 재료를 물어봐야 해. 영어가 안 통하는 식당이 더 많아."

**MenuLens가 주는 해결**:
- 한식진흥원 800선 대조로 **공식 번역**·재료·매운맛 확인
- 온보딩에서 `diet=pescatarian` 한 번 → 모든 응답에서 페르소나 conflict 자동 차단
- 참가격(서울 8품목) 대비 가격 자동 비교 (예: 김밥 평균 ₩3,700)
- D8 PoC 측정: **분식점 80개 메뉴 단일 사진 분석 → 페르소나 conflict 36건 모두 정확 차단** ([proposal.md §3.3](proposal.md))

**사용 시나리오 (D8 실측 데이터 기반)**:
> "서울 연신내역 골목, 분식점 '싸다김밥'. 빨강·검정 메뉴판에 80개 메뉴가 빽빽이. MenuLens 열기 → 찰칵 → 42초 처리 → 🟢 39개·🟡 2개·🔴 36개. 🟢 잔치국수 ₩7,000 / 🟢 해물순두부찌개 ₩8,000 / 🟢 해물볶음밥 ₩8,000 — 어식주의로 안전. 🔴 부대찌개 ₩8,500 (돼지·스팸 포함) / 🔴 돈까스 ₩8,500 (돼지) — 자동 차단. 🟡 치즈김밥 ₩4,500 (김밥 평균 ₩3,700 대비 +22% 가격 주의). 잔치국수 + 해물순두부찌개 카트에 담아 한국어 TTS 재생 후 카운터에 휴대폰 내밂. 주문 완료."

## Persona 3: 말리크 (Malik) — 인도네시아 무슬림 가족, 30대

**배경**: 아내 + 아이 2명. 서울 5일. 모든 식사가 할랄이어야 함. HalalNavi 앱 사용 중.

**현재 경로**:
1. HalalNavi에서 인증 식당만 방문 → 선택지 극히 제한
2. 일반 식당은 "돼지고기 들어있나요?" 매번 물어봐야 함 → 영어 안 통함

**Pain Point**: "아내와 아이들 먹을 수 있는 걸 찾는 게 너무 어려워. 매 끼니가 스트레스."

**MenuLens가 주는 해결**:
- 온보딩에 "할랄" 체크 한 번 → 모든 응답에서 🔴 자동 판정
- 📋 재료 상세: "돼지고기 포함 / 미림(요리술) 의심" 명시
- KMF 인증 식당은 🟢 + 인증 배지 표시

**사용 시나리오**:
> "명동 식당 앞. 메뉴판 찰칵 → 🟢 '생선구이 정식', 🔴 '제육볶음 (돼지고기)', 🔴 '된장찌개 (돼지고기 육수)', 🟢 '비빔밥 (고기빼기 요청 가능)'. 가족과 상의 후 생선구이 3인분 주문. 아이들이 안전하게 먹을 수 있음을 확인."

## Persona 4: 존 (John) — 미국 솔로 여행자, 28세

**배경**: 뉴욕 개발자. 한국어 0%. 기술 사이드 프로젝트로 방한. 편의 최우선.

**현재 경로**:
1. Google Maps → 길찾기 불가
2. ChatGPT에 "recommend Korean food near me" → GPS 없어서 부정확
3. 결국 호텔 근처 Starbucks → 한국 맛을 못 느낌

**Pain Point**: "Korea is high-tech but everything about eating is manual. I don't want to read. I just want it to work."

**MenuLens가 주는 해결**:
- 앱 실행 0.5초 → 카메라 → 찰칵
- 색깔 3종만 보면 됨 → **읽기 0**
- 영어 TTS 없음, 한국어 TTS만 → 오히려 좋음 ("I just hand the phone to the waiter")

**사용 시나리오**:
> "Hongdae alley. Menu all in Korean. Opens MenuLens. Snap. Green menu at ₩9,000 pops up. Tap. Korean audio plays. Hands phone to waiter. Waiter nods, enters order. Food arrives. John eats. Done."

## Cross-cutting Insights

**공통 패턴**:
- 모두 **재방문 or 장기 체류**에서 가치가 커짐
- **설치 저항**이 크므로 PWA/QR 중요
- **언어보다 "무엇을 해야 할지"**가 진짜 pain

**세그먼트별 우선순위 (MVP에서 잡을 것)**:
1. **재방문 FIT** (유이) → 가장 충성도 높음, 제품 완성도 평가 정확
2. **특수 식단** (말리크) → 대체 불가한 가치 제공 가능
3. **1회차 FIT** (천이야) → 최대 시장이지만 설치 장벽 큼
4. **언어 0 초보** (존) → 기능 테스트 최적, 마케팅 소구력 높음

**제안서 1페이지에 녹일 한 문장**:
> "방한 외국인 2,000만 중 재방문자 54.7%, 특수 식단 7%, 언어 장벽 100%에게 MenuLens는 **4초의 결정**을 가능하게 한다." *(2024 외래관광객조사 재방문율 54.7%, [proposal.md §1.1](proposal.md))*

## 제외한 페르소나 (MVP 스코프 밖)

- **단체 패키지 관광객**: 가이드 통역 의존, 개인 앱 수요 낮음
- **장기 유학생**: 이미 한국어 학습 중, 메뉴 불편 약함
- **비즈니스 출장**: 고급 식당·호텔 컨시어지 의존
- **한류 덕후 (K-pop 팬)**: 차년도 K-Scene 기능으로 대응

MVP는 **4개 페르소나 중 2개(재방문 + 특수 식단)에 집중**, 나머지 2개는 Phase 2로.
