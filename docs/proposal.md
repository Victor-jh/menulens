## 1) 서비스 기획 배경 및 필요성

### 1.1 서비스 기획 배경

**2026년은 방한 외국인이 처음으로 2,000만 명을 돌파하는 해**이다. 야놀자리서치 LSTM 예측은 최대 2,126만 명을 전망하고, 한국관광공사는 "2030년 방한 3,000만 조기 달성"을 공식 목표로 선언했다(2026.1 업무보고).

그러나 같은 해, 한국 관광 신뢰의 밑바닥이 드러났다. **광장시장에서는 상인회 내부에서만 3억 원 규모의 바가지 관련 소송**이 벌어졌고(2025.11), **인천공항-인스파이어 K-POP 공연장 구간은 평소 1만 원이 5만 원(5배)**까지 호가되었다(MBC 뉴스데스크). 이재명 대통령이 2025년 9월 국무회의에서 직접 바가지 문제를 경고하는 수준에 이르렀다.

2024 외래관광객 조사에서 **미식 탐방 희망률 68%가 쇼핑 51.5%를 역전**했지만, 그 미식 탐방이 시작되는 "식당 앞에서의 4초"에 외국인은 4가지를 동시에 모른다.

1. 메뉴판에 뭐라고 쓰여 있는지
2. 가격이 적정한지 바가지인지
3. 알레르기·종교·비건 허용 재료인지
4. 어떻게 주문을 하는지

**수요의 정점과 신뢰의 바닥이 같은 해에 만난 이 구조적 모순**이 MenuLens의 출발점이다.

### 1.2 서비스 필요성

기존 도구는 이 4가지 중 **단 하나만** 해결한다.

| 도구 | 해결 | 공백 |
|---|---|---|
| Papago | 번역 | 가격·알레르기·문화 |
| Google Lens | 번역(한국어 약함) | 전부 |
| Visit Korea 앱 | 관광지 안내 | 메뉴 레벨 0 |
| ChatGPT·Perplexity | 대화형 QA | TourAPI 미연동, 환각 37% |

외국인은 4개 앱을 돌려가며 **3~5분을 쓴다**. 식당 앞에서 이만큼 지체되면 결정은 대개 "패스"가 된다. 한국관광공사의 TourAPI 26만 건, 한식진흥원 길라잡이 800선, 한국소비자원 참가격이 모두 공공데이터로 **무료 개방**되어 있음에도, 이 데이터를 통합해 외국인에게 **즉답 가능한 형태**로 제공하는 서비스는 없다.

**2026년은 멀티모달 AI·한식 공공DB·바가지 국가 어젠다가 처음 한 지점에서 만난 해**이다. MenuLens는 이 세 가지 흐름의 교차점에서 "외국인이 메뉴판 앞에서 돌아서지 않게 하는 4초"를 만든다.

## 2) 서비스 개요

### 2.1 기획 서비스 소개 (한 줄)

**한국 식당 앞 외국인이 메뉴판 사진 1장으로 "이 메뉴를 시켜도 될지"를 4초 안에 판단하게 돕는 AI 어시스턴트**

### 2.2 기획 서비스 주요 기능

**핵심 메커니즘**: 메뉴판 사진 1장 → AI 3개 에이전트 병렬 실행 → 🟢🟡🔴 색깔 오버레이 + 한국어 주문 TTS

**3 에이전트 구조**
1. **Menu Reader** — Gemini 2.5 Flash Vision이 메뉴판 이미지를 파싱해 메뉴명·가격 리스트를 JSON으로 추출
2. **Dish Profiler** — Claude Sonnet + pgvector RAG가 한식진흥원 길라잡이 800선을 대조해 공식 번역·재료·알레르기 14종·할랄/비건 여부·문화 한 줄을 반환
3. **Price Sentinel** — 한국소비자원 참가격 서울 기준 8개 외식 품목과 대조해 **🟢 정상가 / 🟡 약간 비쌈 / 🔴 바가지 의심**을 판정

**사용자 경험 흐름**
- 온보딩 60초: 알레르기·종교·언어 한 번 설정 → 영구 적용
- 앱 실행 → 즉시 카메라 뷰 → 찰칵 → 2~3초 처리 → 색깔 박스 오버레이
- 🟢 메뉴 탭 → 한국어 주문 음성 "김치찌개 하나 주세요" 즉시 재생
- 사용자는 휴대폰을 카운터에 내밀면 된다

### 2.3 서비스 차별성

**"읽기를 없앤다"**가 UX의 최종 진화다. 언어·나이·시력·집중력을 뛰어넘는 색깔 기반 결정.

| | Papago | Google Lens | **MenuLens** |
|---|---|---|---|
| 사용자 액션 | 사진 찍고 읽기 | 카메라 대고 읽기 | **탭 1회 + 색깔 보기** |
| 결정 시간 | 30~60초 | 20~40초 | **3~5초** |
| 필요 스킬 | 영어 읽기 | 영어 읽기 | **색깔 구분** |
| 개인화 | 없음 | 없음 | **알레르기·종교·취향 반영** |
| 가격 정보 | 없음 | 없음 | **공공 벤치마크 대조** |
| 문화 맥락 | 없음 | 없음 | **한식진흥원 표준** |

**역대 13회 관광데이터 공모전 수상작 중 동일한 접근은 0건**이다. 특히 TourAPI LOD(Linked Open Data) SPARQL 엔드포인트를 활용한 수상작은 지금까지 없으며, 이는 "데이터 활용 적절성 20점"에서 명확한 차별점이 된다.

## 3) 데이터 활용 방안

### 3.1 활용 예정 한국관광공사 OpenAPI (필수)

**TourAPI 4.0 KorService2** 및 **다국어 6개 서비스** (영·중간체·중번체·일·독·불·러·서)를 주기능에 활용:

| Operation | 용도 |
|---|---|
| `areaBasedList2` | 사용자 현재 위치 기반 식당 기본 정보 |
| `detailIntro2` | 식당 대표 메뉴·영업시간·휴무 |
| `searchFestival2` | 축제·행사 연계 식당 큐레이션 |
| `detailInfo2` | 반복 정보 (운영 특이사항) |

**TourAPI LOD SPARQL** 엔드포인트(`data.visitkorea.or.kr/sparql`)를 **역대 수상작 최초로 활용**해, 관광분류체계(서비스분류코드) 온톨로지를 알레르기 매핑 및 유사 메뉴 추천 보조로 사용.

### 3.2 데이터 활용 방식 (6개 공공데이터 교차)

```
     [TourAPI — 메인 레이어]
      식당 정보 + 다국어 번역
              │
    ┌─────────┼─────────┐
    ▼         ▼         ▼
[한식진흥원]  [참가격]  [식약처]
  표준 번역   8품목     영양성분
  800선       평균가     알레르기
    │         │         │
    └─────────┼─────────┘
              ▼
     [카카오 로컬 API]
      지역 보정 + 위치
              ▼
      [MenuLens 에이전트]
     색깔 🟢🟡🔴 + TTS 응답
```

| 데이터 | 출처 | 역할 |
|---|---|---|
| TourAPI | KTO (data.go.kr/15101578) | **주기능**: 식당 기본·다국어·LOD |
| 한식진흥원 길라잡이 800선 | 한식진흥원 (15129784) | 공식 표기 대조로 오역 방지 |
| 한국소비자원 참가격 외식비 | 한국소비자원 (price.go.kr) | 서울 8품목 평균가 벤치마크 |
| 식약처 식품영양성분DB | 식약처 (15127578) | 알레르기 재료 보조 검증 |
| 카카오 로컬 API | 카카오 | 지역 보정·위치 |
| KMF 할랄 인증 | 한국이슬람교중앙회 | 할랄 식당 크로스체크 |

문체부·KTO 공동조사에 따르면 메뉴 오역의 41.8%가 간판·광고업체, 27.5%가 프랜차이즈 본사에서 발생한다. MenuLens는 **한식진흥원 공식 표기를 1차 소스로 강제 고정**하고, 표기 불일치 시 자동 교정한다. 이는 단순 기술이 아니라 **공공데이터 가치 실현 그 자체**이다.

## 4) 서비스 발전 방향

### 4.1 단계별 로드맵

**Phase 1 (2026.5~10, 공모전 기간)**
- MVP: 서울 기준 한식 8대 품목 + 600개 식당 PoC
- 3개 언어 우선 지원 (영/중/일)
- 웹앱 + PWA 배포

**Phase 2 (2026.11~2027.6)**
- 라이브 AR 카메라 모드 (실시간 오버레이)
- 전국 확장: RTO(부산·경북·강원·제주) 협력 데이터 추가
- 사용자 크라우드 가격 제보 시스템
- 6개 언어로 확장 (영·중간체·중번체·일·독·불)

**Phase 3 (2027.7~)**
- **B2B 식당 대시보드** — 사장님이 메뉴판 1장 사진 등록 → 다국어 자동 번역·알레르기 표기·가격 정상화 배지 QR 발급
- 외국인 대상 예약 대행 기능 (한국 전화번호 없이)
- 한국관광공사 공식 파트너십 추진

### 4.2 기대효과

**사용자 측**
- 외국인 결정 시간 **30~60초 → 3~5초**로 단축
- 바가지 피해 사전 예방으로 **재방문 의사 개선** (2024 외래관광객 조사 재방문율 45% → 목표 50%+)
- 특수 식단(할랄·비건·알레르기) 관광객이 한국에서 **안전하게 외식**할 수 있는 인프라

**정책 측**
- 방한 3,000만 달성(2030)을 위한 **외국인 신뢰 인프라** 제공
- 공공데이터 6종의 **통합 가치 실현** 사례
- 소상공인 식당의 외국어 대응 부담을 **AI가 대신 처리**

**사회적 가치**
- 메뉴 오역 41.8%(문체부·KTO 조사)를 기술로 해결
- "외국인 바가지"라는 국가 어젠다에 **시민 참여형 해법** 제시
- 공공 API 생태계의 **민간 활용 모범 사례**

### 4.3 지속가능성

**수익 모델**
- **Phase 1~2**: 무료 공공 서비스 (공공데이터 기반 사회적 가치)
- **Phase 3**: B2B 식당 대시보드 월 구독 (₩9,900~29,900), 관광공사 공식 파트너십 라이선스
- 1회성 Premium: 여행 기간 7일 $4.99 (외국인 대상)

**기술 지속가능성**
- 오픈소스 Git 저장소로 공개 (GitHub: Victor-jh/menulens)
- 공공데이터 기반 아키텍처로 **벤더 락인 0**
- Claude·Gemini 이중화로 단일 AI 의존성 제거
