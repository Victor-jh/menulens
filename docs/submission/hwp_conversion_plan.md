## 0. ⚠️ 가장 중요한 사실 — 2026-04-23 업데이트

공식 공고문(`docs/submission/official/2026_공고문.pdf`) p.10 및 HWP 양식 첫 페이지 규정란에 명시된 사항:

> **"작성항목 및 양식 수정 금지, 모든 항목 작성 필수"**

이에 따라 **기존에 기획한 `Pandoc MD → DOCX → HWP` 변환 파이프라인은 폐기**한다. Pandoc으로 새 HWP를 생성하면 공식 양식의 표 구조를 재현할 수 없어 "양식 수정" 위반 위험이 있다.

### 새 원칙

1. **공식 HWP 양식 원본을 한컴오피스로 열고 각 표 셀에 직접 내용을 붙여넣는다.**
2. **제출 포맷은 PDF(10MB 미만)이며, HWP는 편집용이다.** 따라서 작성 완료된 HWP에서 "다른 이름으로 저장 → PDF"로 최종 산출.
3. `docs/proposal.md`는 **내용의 소스 오브 트루스**로 유지. HWP는 최종 표현 레이어.

## 1. 공식 양식 구조 (HWP 원본 기준, 2026-04-23 추출 확인)

```
[표 1] 제목 박스
  └ 『2026 관광데이터 활용 공모전』- ① 웹·앱 개발 부문 제안서

[표 2] 규정 박스
  └ 작성항목 및 양식 수정 금지, 모든 항목 작성 필수, 이미지(사진), 표 등 삽입 가능
  └ 분량: 5페이지 이내로 작성 (이미지, 표 등 일체 포함)
  └ 글자 포인트는 12포인트 이상
  └ 제출파일 형식: PDF (10MB 미만)

[표 3] 1) 서비스 기획배경 및 필요성
  ※ 서비스를 기획·구상하고 제안한 배경과 필요성 작성
  └ 서비스 기획 배경 [빈 셀 — 여기에 작성]
  └ 서비스 필요성 [빈 셀]

[표 4] 2) 서비스 개요
  ※ 기획 서비스에 대한 소개 및 구현 예정인 주요 기능 위주로 작성
  ※ 지역 특화 서비스의 경우, 특화 지역 관련 내용 작성 필수
  └ 1. 기획 서비스 소개 (한 줄 작성): ex) 인구 감소지역 관광정보 제공 서비스
  └ 2. 기획 서비스 주요 기능: ex) 관광정보 검색 기능, 맞춤형 코스 추천 기능
  └ 3. 서비스 차별성
  └ 4. (지역 특화 서비스의 경우 작성 필수) 서비스 내 지역 특화 관련 사항 [MenuLens 해당 없음 → 공란 또는 "해당 없음 (전국형 서비스)" 기재]

[표 5] 3) 데이터 활용 방안 *한국관광공사 OpenAPI 활용 필수
  └ 1. 활용 예정 한국관광공사 OpenAPI: ex) 한국관광공사 국문관광정보 서비스
  └ 2. 데이터 활용 방식

[표 6] 4) 서비스 발전 방향
  ※ 기획 서비스 발전 가능성, 기대효과 등에 대해 작성
  └ 1. 개발 서비스 향후 발전방향
```

**주의**: 양식에는 "지역특화 관련 사항" 필드가 있음. MenuLens는 전국형 서비스라 `"해당 없음 (전국형 서비스로 지역특화 해당 없음)"`으로 1줄만 기재.

## 2. D11 실행 워크플로우 (약 1.5~2시간)

### Step 1 — 준비 (10분)

### Step 2 — 섹션별 작성 (60~90분)
proposal.md의 각 섹션을 HWP 양식의 해당 빈 셀에 **복사·붙여넣기**한다. 순서:

| HWP 셀 | proposal.md 출처 | 주의사항 |
|---|---|---|
| 1) - 서비스 기획 배경 | §1.1 | 방한 2,000만·광장시장 3억원 등 출처 각주 포함 |
| 1) - 서비스 필요성 | §1.2 | 기존 도구 비교 표 여기 삽입 |
| 2) - 1. 기획 서비스 소개 | §2.1 (1줄) | "찰칵 한 번, 초록색만 주문하세요 — 한국 식당 앞 외국인 메뉴판 AI 어시스턴트" 등 한 줄 압축 |
| 2) - 2. 주요 기능 | §2.2 | 3 에이전트 · 찰칵 · 색깔 3종 · 탭 주문 |
| 2) - 3. 차별성 | §2.3 | 비교 표 + "읽지 않는 UX" |
| 2) - 4. 지역특화 | (없음) | "해당 없음 (전국형 서비스)" 1줄만 |
| 3) - 1. 활용 예정 OpenAPI | §3.1 | TourAPI 4.0 · 다국어 6개 · LOD SPARQL |

**복붙 유의사항**:
- Markdown 문법(`**bold**`, `# header`) 제거 필수 (한컴에서 깨짐)
- 표는 Markdown에서 HWP 표로 **수동 변환** (행·열 복사 또는 Ctrl+C→Ctrl+V 시도)
- 이미지는 별도로 삽입 (§2.2·§3.2 다이어그램·시연 스크린샷)
- 줄바꿈은 HWP 에디터에서 자연스러운 단락 단위로 재정렬

### Step 3 — 형식 점검 (15분)
- [ ] 전체 분량 **5페이지 이내** (페이지 하단 번호 확인)

## 3. PDF 크기 압축 (10MB 초과 시만)

### 3.1 macOS Preview.app
1. Preview로 PDF 열기
2. 파일 → 내보내기 → Quartz 필터: `Reduce File Size` 선택
3. 저장 → 파일 크기 재확인

### 3.2 ghostscript (CLI, 품질 정밀 제어)
```bash
brew install ghostscript
gs -sDEVICE=pdfwrite -dCompatibilityLevel=1.4 \
   -dPDFSETTINGS=/printer \
   -dNOPAUSE -dQUIET -dBATCH \
   -sOutputFile=MenuLens_제안서_최종_압축.pdf \
   MenuLens_제안서_최종.pdf
```
`/printer` → 300dpi 유지 · `/ebook` → 150dpi (이미지 많을 때만)

### 3.3 ilovepdf.com 등 온라인 서비스
- **권장 안 함** (제안서 유출 리스크)

## 5. Pandoc 기반 기존 플랜 폐기 사유 (이력)

**기존 (D3 초안, 폐기)**: `proposal.md` → `pandoc` → `proposal.docx` → 한컴오피스 → `proposal.hwp` → PDF

**폐기 이유**:
1. Pandoc은 새 HWP/DOCX를 생성하므로 공식 양식의 **표 구조·셀 구성을 재현할 수 없다**
2. "양식 수정 금지" 규정 위반 가능성. 심사위원 재량에 따라 **제출서류 미비로 실격 위험**
3. 양식 원본에 직접 붙여넣는 것이 오히려 **빠르고 안전**

**새 플랜 (D3 재작성)**: `proposal.md` = 소스 → **HWP 양식 원본 복사 → 빈 셀에 붙여넣기 → PDF 저장**

## 6. 폰트·여백·이미지 기준 (참고)

### 6.1 폰트
- **본문**: 맑은 고딕 또는 함초롱바탕 **12pt 이상** (공식 필수)
- **제목·부제**: 공식 양식 그대로 유지 (수정 금지)
- **영문**: Arial 또는 Helvetica 12pt (한글과 섞일 때)
- **폰트 임베드 필수**: PDF 저장 시 "글꼴 포함" 옵션 반드시 체크

### 6.2 여백
- **공식 양식 그대로 유지** (수정 금지). 사용자가 임의 변경하지 않는다.

### 6.3 이미지
- 해상도 150dpi 이상, 권장 300dpi
- PNG(다이어그램) · JPG(사진) 혼용 가능
- 저작권 확인 (Pixabay CC0, Unsplash 라이선스 등)
- **본인이 찍은 메뉴판 사진**을 §2.2 시연 이미지로 삽입 권장

## 7. 자주 발생하는 문제 & 해결

| 증상 | 원인 | 해결 |
|---|---|---|
| 5페이지 초과 | 텍스트 과다 | §1.1·§4.3 기대효과 등 2~3문장 축약 |
| 10MB 초과 | 이미지 해상도 과다 | §3.2 ghostscript `/printer` 재압축 |
| 한글 폰트 깨짐 (PDF) | 폰트 미임베드 | PDF 저장 옵션 "글꼴 포함" 재확인 |
| 표 셀 줄바꿈 문제 | 복붙 시 서식 충돌 | Ctrl+Shift+V (서식 없이 붙여넣기) 사용 |
| 이미지가 셀 밖으로 튀어나옴 | 이미지 크기 셀보다 큼 | 이미지 속성 → "글자처럼 취급" + 크기 조정 |
| 목차가 자동 생성 안 됨 | 한컴에서 목차는 필요 없음 (5p 제안서) | 목차 생략 |
| 각주 번호가 엉망 | Markdown 각주 → HWP 자동 전환 불가 | 각주는 본문 중 괄호 내 "(출처: ..)" 형식으로 수동 삽입 권장 |
